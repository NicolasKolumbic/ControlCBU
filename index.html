Control AngularJS CBU para PISCYS

Objetivo:Permitir la entrada de CBU valido.

Desarrollado con los mismos criterios de los campos en AS DIRECT con HTML, CSS, AngularJS y Javascript.

Código: Javascript de Vista dentro del controlador
        $scope.Datos = {
            NumeroCBUuno: "2850590940090418135201",
            NumeroCBUunoError: "2850590940090418135205" 
        }
Código: HTML de Vista
    <control-cbu id="ComponentCBUuno" model="Datos.NumeroCBUuno" label="CBU"></control-cbu>

Las propiedades obligatorias son model e id. El primero representa la relacion de la vista con el controlador Angular.js, la propiedad ng-model. La segunda es un id que identifica al control, es importante en caso que se encuentre mas de un control de este tipo en la vista y una mejora con respecto a los controles de AS DIRECT que automaticamente se genera un objeto en el $scope padre con el id provisto por la propiedad.

la propiedad label genera un etiqueta para el control con el string pasado por valor.

Instancia del Objeto CbuControl en el $Scope

Una de las cuestiones que sucedia en AS DIRECT era la imposibilidad de cambiar un estado dentro del controlador Angular.js, o mejor dicho. Los controles desarrollados no tiene tales opciones. Por ejemplo dependiendo de una configuración se desactive o active un control. Esta fue una mejora a plantear

Como comente anteriormente cada vez que se crea una instancia del control control-cbu se génera una instancia de la clase Javascript cbuControl dentro del $scope que contiene el control. 

    // Desactiva el control.
    $scope.ComponentCBUdos.Disabled(true);
    

    // Activa el control.
    $scope.ComponentCBUdos.Disabled(false);

        NumeroCBUdos:""

    $scope.Activar = function(){
        $scope.ComponentCBUdos.Disabled(false);     
    }

    $scope.Desactivar = function(){
        $scope.ComponentCBUdos.Disabled(true);     
    }

En el ejemplo se utilizan dos botones para Activar y Desactivar el campo de entrada de CBU. Los botones utilizan la directiva ng-click para vincular los eventos a los botones. Los métodos se encuentran en la solapa Javascript junto al ng-model.

Propiedades

required

Define el campo como requerido. Equivale a la propiedad HTML required. Al definir esta propiedad como verdadera cada vez que sea haga foco(focus) y luego se salga(blur) ejecutara un mensaje de error. Cuando un campo es obligatorio tiene un fondo amarrillo palido como AS DIRECT.

<div>
  <control-cbu id="ComponentCBUcuatro" model="Datos.NumeroCBUcuatro" label="CBU" required="true" ></control-cbu>
</div>

setdisabled

Define el campo como desactivado. Equivale a la propiedad HTML disabled. Al definir esta propiedad como verdadera el control toma de color de fondo un color gris y el cursor cambia evitando que se pueda editar el valor.

<div>
  <control-cbu id="ComponentCBUcuatro" model="Datos.NumeroCBUcuatro" label="CBU" setdisabled="true" ></control-cbu>
</div>

cssclass

Define una clase css para el campo. En la estructura interna del control; este esta formado por una serie de divs, un label, spans y un input al que se le escribirá el atributo.

<div>
  <control-cbu id="ComponentCBUsexto" model="Datos.NumeroCBUsexto" label="CBU" cssclass="blurControl" ></control-cbu>
</div>
.blurControl{
background-color: #b7cce680 !important;
border:2px solid skyblue;
}

ng-class

Equivale al atributo Angular.js ng-class al cual se le puede pasar por valor un objeto JS que determine una clase css.

<div>
  <control-cbu id="ComponentCBUseptimo" model="Datos.NumeroCBUseptimo" label="CBU" ng-class="{blurControl:error}" ></control-cbu>
</div>

readonly

Define un campo de solo lectura

<div>
  <control-cbu id="ComponentCBUoctavo" model="Datos.NumeroCBUoctavo" label="CBU" readonly="true" ></control-cbu>
</div>

Otros métodos

$scope.[Instance].Required(boolean)

Permite determinar como requerido o no un control CBU. Equivale a la propiedad required de un control input html 

    // Define el campo como requerido
    $scope.ComponentCBUdos.Required(true);

    // Define el campo como no requerido
    $scope.ComponentCBUdos.Required(false);
$scope.[Instance].Disabled(boolean)

Permite desactivar/activar un control CBU. Equivale a la propiedad disabled de un control input html 

    // Define el campo como requerido
    $scope.ComponentCBUdos.Required(true);

    // Define el campo como no requerido
    $scope.ComponentCBUdos.Required(false);
$scope.[Instance].IsDisabled()
$scope.[Instance].IsRequired()

Ambos métodos obtienen el valor booleano de la propiedades de disabled o required en cada caso. 

    // Devuelve true si el campo esta desactivado
    $scope.ComponentCBUdos.IsDisabled();

    // Devuelve true si el campo esta definido como requerido
    $scope.ComponentCBUdos.IsRequired();
$scope.[Instance].AddClass(str)
$scope.[Instance].RemoveClass(str)

Ambos métodos modifican el atributo HTML class.AddClass permite agregar una clase CSS al control pasandola por parametro al método. RemoveClass elimina la clase pasada por parametro.

Eventos

on-blur

Se ejecuta al perder el foco el control (blur). Recibe dos parametros: objeto JQuery para manipular el DOM y el cbu.

  • Vista
  • HTML
  • Javascript

<control-cbuid="ComponentCBUnoveno" model="Datos.NumeroCBUnoveno" required="true" label="CBU"
on-blur="FormatearCBU"></control-cbu>
<p>
<label>CBU formatedo personalizado:</label>
<span id="tagCbu"></span>
</p>
 
    $scope.Datos = {
        NumeroCBUnoveno:""
    };

    $scope.FormatearCBU = function(elm,cbu){
        var txt = "";
        txt = cbu.split('').reverse().join('').replace(/(?=\d*\.?)(\d{3})/g, '$1 ');
        txt = txt.split('').reverse().join('').replace(/^[\.]/, '');
        $("#tagCbu").text(txt);
    }
